Hypothesis

Without sanitization, raw HTML tags pass through to the response, enabling injection attacks.

user_text = sanitize_html(user_text)
Experiments
Experiment A: Strip only script tags

Remove only <script> but keep other HTML

cleaned = re.sub(r'', '', text, flags=re.DOTALL)

Observe: Blocks scripts but allows <img onerror=...> and <a href=javascript:...>.

Experiment B: HTML-encode instead

Convert HTML chars to entities instead of stripping

import html; cleaned = html.escape(text)

Observe: Safe but changes display. User sees &lt;b&gt; instead of bold.

Experiment C: Sanitize first 100 chars only

Only clean the beginning of the message

cleaned = sanitize_html(text[:100]) + text[100:]

Observe: Attacker puts payload after character 100 to bypass sanitization.

#13 Remove Message Length Check Input Bounds Checking ▾
Hypothesis

Without length limits, arbitrarily long messages are accepted, enabling resource exhaustion attacks.

Action

app/validation.py (lines 30-31)

Comment out the length check block.

Experiments
Experiment A: Limit to 10 chars

Set maximum length to 10 characters

if len(message) > 10: raise ValueError('Too long')

Observe: Nearly all legitimate messages are rejected. The system is technically secure but unusable.

Experiment B: Limit to 1 million chars

Set a very generous limit

if len(message) > 1_000_000: raise ValueError('Too long')

Observe: 1MB of text per message. Enough to overwhelm the LLM’s context window and spike API costs.

Experiment C: Warn but don’t reject

Log a warning for long messages but process them anyway

if len(message) > MAX_LEN: logger.warning('Long message')

Observe: Creates audit trail but doesn’t prevent resource exhaustion. Useful for monitoring, not protection.

Increment counter but with hardcoded label

Only increment on exceptions

Experiment C: Use gauge instead

Replace counter with gauge

TOOL_CALL_GAUGE.labels(tool=name).set(1)

Observe: Gauge shows last-call status, not cumulative count. Rate calculations become impossible.

Return a plain string instead of structured error

except Exception: return "An error occurred"

Observe: LLM gets a response but no details. It can’t explain to the user what went wrong or how to fix it.
