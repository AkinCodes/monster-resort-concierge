#18 Remove RRF BM25 Weight Reciprocal Rank Fusion NO TEST ▼
Hypothesis

Without BM25 scores in RRF, fusion becomes dense-only, losing keyword matching capability. Queries that depend on exact keyword matches will return significantly worse results.

Action

File: app/advanced_rag.py (lines 188–190)

Comment out the BM25 weight loop in the RRF calculation:

# for doc_id, bm25_score in bm25_results:
# rrf_scores[doc_id] += bm25_weight / (k + bm25_rank)
# bm25_rank += 1
Experiments
Experiment A: BM25 weight = 0.9

Make BM25 dominant over dense retrieval

bm25_weight = 0.9; dense_weight = 0.1

Observe: Keyword-heavy queries improve dramatically. Semantic queries degrade. Proper nouns and exact terms dominate results.

Experiment B: BM25 weight = 0.1

Make dense retrieval dominant

bm25_weight = 0.1; dense_weight = 0.9

Observe: Semantic queries improve. Keyword queries for specific entity names (e.g., 'Vampire Manor') return less precise results.

Experiment C: Simple averaging instead of RRF

Replace RRF with arithmetic mean of scores

combined = {d: (bm25.get(d,0) + dense.get(d,0))/2 for d in all_docs}

Observe: Loses RRF's rank-based normalization. Raw score differences between BM25 and dense create bias toward whichever produces larger numbers.

Convert query to uppercase instead of lowercase
